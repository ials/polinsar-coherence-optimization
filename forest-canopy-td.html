Tropical Forest Canopy Height — Temporal Decorrelation Models

Reproduces the core idea of Luo et al. (2024): repeat-pass PolInSAR coherence is corrupted by temporal decorrelation (TD) — mostly wind-driven canopy motion — which the plain RVoG model can't see, so it compensates by inferring a taller tree than actually exists. The paper compares four "stage-3" inversion recipes (RVoG, RMoG, RVoG+VTD, and its own faster RVoG+VTDs) that estimate canopy height from the same noisy, TD-corrupted coherence — this dashboard runs all four live, using the paper's own algorithms.

Source: H. Luo, C. Yue, Y. Wu, X. Zhang, C. Lu, G. Ou, "An efficient method for estimating tropical forest canopy height from airborne PolInSAR data," Ecological Indicators 166 (2024) 112566 ("forest canopy height.pdf" provided by the user).

5 — Fig.10 spirit: height of ambiguity and kz vs. canopy height

Hoa=2π/kz is the height beyond which the interferometric phase wraps. The paper's PROD baseline-selection method tends to pick smaller kz (larger Hoa) for taller canopies to stay unambiguous — but smaller kz also means less height sensitivity, which the paper offers as one explanation for growing scatter above ~45 m. The illustrative kz(hv) curve here follows that same trade-off logic, not the paper's real baseline-selection data.

RVoG volume coherence γV uses the standard closed form (consistent across this dashboard series). Stage-3 "already ground-separated" pure-volume coherence with TD: RVoG (no compensation) γh=eiφ₀γV(hv,σ); RVoG+VTD/VTDs γh=eiφ₀γTVγV(hv,σfixed) (Eq. 17/19, σfixed = true σ + the estimation-error slider); RMoG (μ=0, σg=0 per the paper's own simplification) γh=eiφ₀·p₁[e(p₂+p₃)h_v−1]/[(p₂+p₃)(ep₁h_v−1)], p₁=2σfixed/cosθ, p₂=p₁+ikz, p₃=−σv²/(2hr)·(4π/λ)², λ=0.2384 m (UAVSAR L-band), hr=1 m (paper notes results are insensitive to this reference height) (Eq. 14). RVoG+VTDs solves hv in a single loop, computing γTV algebraically at each trial height (Eq. 19) rather than grid-searching it — this dashboard's own iteration counter in panel 1 demonstrates the resulting ~N× speed-up directly, mirroring the paper's reported ≈22× figure. Panel 3 quotes the paper's real Table 2 / Section 4.2 numbers verbatim (a very minor 1-minute discrepancy exists between the paper's own abstract and body text for RMoG's runtime — 28 vs. 29 min — the body-text figure is used here). Panels 2, 4 and 5 are illustrative live simulations built to match the paper's qualitative and, where stated, quantitative findings — not reproductions of the real AfriSAR/LiDAR dataset.
